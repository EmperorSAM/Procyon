Russian CSS Framework

Скачать

Процион — OpenSourse CSS Framework разработанный в России.

Процион создавался как лёгкий и гибкий инструмент для Front-end разработки. Можно не использовать framework целиком, если в этом нет необходимости. Для этого весь код был разбит на компоненты.

Procyon работает на jQuery 1.7 (входит в состав дистрибутива).

Для разработчиков на Drupal есть дополнительная информация.

Страница проекта на GitHub.

Адаптивная сетка

Допустим, Вы указали элементу класс sm-6. То, для Планшета и экранов больше, этот элемент будет занимать 6 колонок. А для Смартфона, то есть экранов меньше, он будет занимать все колонки.

Можно указывать сразу несколько классов для различных типов экранов.

.sm-2 .xs-4
.sm-6 .xs-4

100% .sm-8 .md-4
100% .md-2
100% .sm-4 .md-5 .lg-7
100% .sm-8 .md-5 .lg-3
100% .md-2 .lg-2

Контент по середине страницы

Чтобы материал на странице был по середине экрана, поместите его в <div class="container"> ... </div>. Этот элемент будет менять ширину в зависимости от типа экрана. Как на этом сайте.

Размеры центрирования:

  • Мобильные - 100% (Резиновая)
  • Планшеты (от 768px до 991px) - 768px
  • Средние экраны (от 992px до 1199px) - 992px
  • Большие экраны (от 1200px и выше) - 1200px
HTML
<body>
    <div class="container [fluid-*]">...</div>
    <section>
        <div class="container [fluid-*]">...</div>
    </section>
</body>

Выборочная резиновая вёрстка

Если Вы хотите использовать "резиновую" верстку, то просто не используйте данный класс вообще. Но если вдруг понадобилось переключаться на резиновую верстку для некоторых размеров экранов, то существую следующие классы. Их необходимо добавить в элемент, пример <div class="container fluid-md-up"> ... </div>.

  • .fluid-sm - "резиновый" container, только для планшетов.
  • .fluid-sm-up - для планшетов и больше.
  • .fluid-sm-down - для планшетов и меньше.
  • .fluid-md - только для средних дисплеев.
  • .fluid-md-up - для средних дисплеев и больше.
  • .fluid-md-down - для средних дисплеев и меньше.
  • .fluid-lg - только для больших дисплеев.
  • .fluid-landscape - для альбомной ориентации
  • .fluid-portrait - для портретной ориентации
  • .fluid-4by3 - для диагонали дисплеев 4/3
  • .fluid-16by9 - для диагонали дисплеев 16/9
  • .fluid-16by10 - для диагонали дисплеев 16/10
  • .fluid-16by9and10 - для диагонали дисплеев 16/9 и 16/10
  • .fluid-print - для режима печати

Вложенные .row

  • .row > .row (прямой потомок) - у второго .row отступа не будет. Можно добавлять другие блоки.
  • [xs-# sm-# md-# ld-# block-*] > .row (прямое наследование) - у второго .row появляются отрицательные левые и правые отступы для выравнивания. Поэтому его надо помещать непосредственно в адаптивные ячейки .xs-#.

Режимы показа. Где * вышеперечисленные классы.

  • * (без дополнений, по умолчанию) - {display: inherit !important;}
  • *-block {display: block !important;}
  • *-inline {display: inline !important;}
  • *-inline-block {display: inline-block !important;}

Пример:

  • .show-xs-inline - Показать в линию для всех дисплеев.
  • .show-print-inline - Показать в линию в режиме печати.
  • .show-4by3-inline - Показать в линию для мониторов диагональю 4/3.
  • .show-sm-up-block - Показать для планшетов и выше в режиме block.

Кнопки

Все классы кнопок строятся следующим образом .button *, где * Ваш класс. Таким образом кнопка будет получать стандартные стили для кнопок + Ваши собственные стили.

В .base и .border используется базовый цвет, который можно установить в основном файле.

Для элемента button по умолчанию применяются общие стили форматирования от .button. Далее Вы можете сами дописать необходимые стили или примешать готовый стиль через less.

Для всех кнопок можно использовать следующие дополнительные классы (размеры и стили можно использовать одновременно):

  • Размеры:
  • .big - Большая кнопка.
  • .small - Маленькая кнопка.
  • Стили:
  • .expand - На всю ширину элемента.
  • .radius - Закруглённые углы (по общему радиусу в настройках).
  • .round - Закруглённые углы до уровня круга.
  • .disabled - Деактивировать кнопку (можно не классом, а атрибутом).

$(document).ready(function () {

Модальные окна

Открыть модальное окно

  • Управление
  • HTML

Чтобы открыть модальное окно, необходимо к элементу добавить data-modal-id="myModal", где myModal - это id открываемого модального окна.

Закрыть модальное окно можно на клавишу ESC или нажатием на фон модального окна.

<!-- HTML -->
<a data-modal-id="myModal">Открыть модальное окно</a>
<a data-modal-close>Закрыть модальное окно</a> <!-- Кнопка может находиться где угодно -->
// JS

// Через событие
$('a.modal-link').triger('click');
$('a[data-modal-close]').triger('click');
                    
// Вызвать или закрыть модальное окно
$('#myModal').modal();
$('#myModal').modal({keyboard: false}); // Отключить клавишу ESC для модального окна
<div id="myModal" class="[modal-xs modal-sm modal-md modal-lg modal-full]" data-modal>
    <h2>Модальное окно</h2>
    <p>Это модальное окно.</p>
    <a data-modal-close>Закрыть</a> <!-- Кнопка закрытия -->
    <span data-modal-close></span> <!-- Крестик сверху-справа для закрытия -->
</div>

Размеры окна

Класс размера окна добавляется к основному контейнеру модального окна. Пример <div id="myModal" class="modal-xs" data-modal>.

Если не задавать определенный размер, то ширина составит 80%.

Все модальные окна на маленьких экранах (смартфонах) имеют 95% ширины, кроме .modal-full.

  • .modal-xs - ширина 30%
  • .modal-sm - ширина 40%
  • .modal-md - ширина 60%
  • .modal-lg - ширина 90%
  • .modal-full - на весь экран

Элементы img и embed по умолчанию адаптивны, то есть они занимают все доступное им в блоке пространство. Так что просто поместите картинку в блок необходимого размера.

Дополнительные классы для темизации изображения: .round, .radius, .circle.

<img class="[round radius circle]" />
<embed class="[round radius circle]" />

Less

В исходном файле procyon.less Вы можете сменить базовые свойства через переменные. А так же включить или отключить нужные компоненты, путём комментирования @import. Все компоненты лежат в папке less.

Так же присутствуют множество классов для примешивания. Они начинаются с .to-*.

Используемые стандартны написания кода.

Свойства и классы

z-index:

  • 1000 - .topbar.fixed-top
  • 1003 - [data-superbar]
  • 1005 - .modal-bg
  • 1006 - [data-modal]
  • 1010 - .back-to-top
  • 2000 - .do-not-go

Классы расширений к компонентам (все):

  • .open - Элемент открыт.
  • .active - Элемент активен.
  • .collapsible - Сложенный изначально элемент.
  • .dropdown - Элемент с выпадающим вниз другим элементом. Или сам выпадающий элемент.
  • .small - Маленький элемент.
  • .big - Большой элемент.
  • .radius - Со скруглёнными углами.
  • .round - Со скруглёнными до круга углами.
  • .base - Дизайн на базовом цвете.
  • .dark - Дизайн в тёмных тонах.
  • .minimal - Прозразный дизайн c серыми вставками.
